Detailed Description

Generalized Schur Algorithm for arbitrary size N.

The implementation of Generalized Schur algorithm for any N contains following steps:

1) Decompose integer N into the binary form w.r.t modulus b and obtain a vector s s.t. \(s = {2^{k_0} \times b, 2^{k_1} \times b, ..., 2^{k_T} \times b, r}\), where \(k_0 > ... > k_T \geq 0\), \(r < b\), and sum(s) = x. This step is done with function int2Bin. As in GSchur Algorithm, whole size-N transformation \(T_{0,N}\) can be decomposed into binary pieces in following way

\[ T_{0,N} = T_{0, s[1]} \circ T_{s[1], s[2]} \circ T_{s[1]+s[2], s[2]} \circ ... \circ T_{s[1]+s[2]+...s[m], r} \]

2) Use doubling procedure to compute the binary pieces \(T_{s[1]+s[2]+..., s[i]}\). This step is done with function GenStep of GSchurN class.

3) Merge the binary pieces in ascending order (for computation efficiency) and finally obtain the size-N transformation \(T_{0,N}\). This step is done with function GenMerge of GSchurN class.

Progress function of Generalized Schur Algorithm that computes \( \{\alpha_{n,n} \beta_{n,n} \} \) from \( \alpha_{0,n}, \beta_{0,n}, \eta_{n,n}, \xi_{n,n} \), corresponds to Step (5) in Algorithm (1)

For given length-N polynomials \(\alpha_{0,n}(x), \beta_{0,n}(x), \eta_{0,n}(x), \xi_{0,n}(x)\), compute length-N polynomials \( \alpha_{n,n}, \beta_{n,n} \) defined as

\[ \alpha_n = (\alpha_0 \eta_n - \beta_0 \xi_n) / x^n, \beta_n = (\beta_0 \tilde\eta_n - \alpha_0 \tilde\xi_n) / x^n \]

where \( \tilde\eta_n = x^n \eta_n(1/x) \)

Trick for computation efficiency: by negating the odd parts of the conjugation of fft( \(\tilde\eta_n, 0, 0, ...\)), we can directly get fft( \(0, \tilde\eta_n, 0, ...\)).

Merge the binary pieces \(T_{a,b}\) to get \(T_{0,N}\).

Compute the binary pieces \(T_{a,b}\) from s. Size of such pieces is power of 2 times modulus b.

This function implements the recursive doubling procedure in Generalized Schur Algorithm. It corresponds to the Step (4) - (9) in Algorithm 1. For an input size si, function use memory gs[0], ..., gs[layer] to compute polynomials \( \eta_{0,s[i]}, xi_{0,s[i]} and \gamma_{s[i]} \) from \( \alpha_{0,s[i]}, \beta_{0,s[i]}\). Size si is an element of vector s, which records the binary decomposition of N. As a result si is either \(\leq b\) (where layer = 0) or \(= 2^{layer} \times b\).

Parameters

[in]beta0Second input real vector.

[in]layerIndex of address for computation, i.e. computation happens in gs[0,1,..., layer]

polynomials \( \{ \eta_{0,s_i}, \xi_{0,s_i}, \gamma_{0,s_i}\} \) stored in gs[layer]->eta_0_IFFT->out, gs[layer]->xi_0_IFFT->out, gs[layer]->gamma.

Progressive Schur Algorithm for arbitrary size.

It is an \(O(n^2)\) algorithm. Numerical experiments shows that for \(N\leq 64\) PSchur Algorithm is faster than GSchur algorithm. Thus we choose binary modulus to be 64 in implementation, i.e. computation of pieces smaller than 64 is done with PSchur and the merging of such pieces are done with GSchur.

Parameters

[in]beta0Input real vector.
[in]nSize of inputs (integer).

Vector of GSchur2K for the binary pieces \( T_{0, s[1]} < ,T_{s[1], s[2]},...\). Because of the doubling procedure, the computation of \( T_{, 2^k\times b}\) requires GSchur2K of size \((b, 2\times b, ..., 2^k\times b)\).
